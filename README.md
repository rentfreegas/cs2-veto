# CS2 Veto Predictor

**Live app: https://rentfreegas.github.io/cs2-veto/**

> **Work in progress** — this is an early version of the model, trained on a
> partial match history. It will be retrained on a much larger dataset within
> the next few weeks.

Predicts the map veto for any professional Counter-Strike 2 matchup — each
team's expected bans and picks, the most likely full veto sequence, and the
probability that each map gets played.

## How it works

Pro teams are creatures of habit: two thirds of them ban the same map first
in at least 8 of their last 10 vetos. This app models the veto as a sequence
of choices — at every step, a model scores each remaining map from both
teams' recent veto tendencies (recency-weighted ban/pick rates, map form,
and what the *opponent* is likely to want) and then walks every possible
veto path to produce exact probabilities.

Everything runs **in your browser**. The model weights and team profiles
(500+ teams, built from public veto records) are baked into a single HTML
file. No backend, no accounts, no tracking, no cookies.

## Track record (graded live)

The page carries its own scoreboard. Every day the pipeline logs predictions
for upcoming pro matches *before they start* (append-only log), then grades
them against the veto that actually happened once results land. The
**Track record** section on the page shows the live hit rates and every
graded match — what we predicted, what actually happened, and exactly what
changed when we missed. Predictions logged after a match started are voided,
never scored.

## Accuracy (held-out backtest)

Measured on 2,194 held-out pro matches (Apr–Jul 2026) the model had never
seen, with the acting order unknown (true pre-match conditions):

| What | Hit rate |
|---|---|
| A team's first ban | 81% |
| A team's map pick | 55% |
| At least 2 of the 3 maps played in a BO3 | 86% |
| All 3 maps exactly | 32% |
| All 3 played maps within its top-4 | 62% |

(Random guessing gets a first ban right 14% of the time.)

## Deployment

The site is a single static `index.html` (plus `_headers` for Cloudflare) —
no build step. It is generated by `scripts/export_veto_web.py` in the main
prediction project and copied here.

**GitHub Pages** (current): serves this repo's root at the URL above.
`.nojekyll` keeps Jekyll out of the way.

**Cloudflare Pages** (planned move):

1. Cloudflare dashboard → **Workers & Pages → Create → Pages →
   Connect to Git** and pick this repo (`rentfreegas/cs2-veto`).
2. Project name: whatever the site should be called, e.g.
   `cs2esportsvetopredictor` (becomes
   `cs2esportsvetopredictor.pages.dev`).
3. Build settings: framework preset **None**, build command **empty**,
   output directory **/** (repo root). Save and deploy.
4. Every `git push` to `master` then auto-deploys to both GitHub Pages and
   Cloudflare Pages — nothing else changes.
5. Custom domain (optional): Pages project → **Custom domains → Add**, and
   follow the DNS prompt (instant if the domain is already on Cloudflare).

The `_headers` file in this repo is picked up by Cloudflare Pages
automatically (GitHub Pages ignores it).
